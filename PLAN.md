# RIVALS Web — 개발 계획

로블록스 [RIVALS](https://www.roblox.com/games/17625359962/RIVALS)에서 영감을 받은 웹 기반 1인칭 슈터.
브라우저에서 바로 플레이 가능하며, 봇 대전(오프라인)과 온라인 1v1 멀티플레이를 모두 지원하는 것이 목표.

## 확정된 방향

| 항목 | 결정 |
|---|---|
| 플레이 방식 | 봇 대전 우선 구현 → 온라인 멀티(방 코드 기반 1v1) 추가 |
| 첫 게임 모드 | 1v1 듀얼 (라운드제, 5라운드 선취 승리) |
| 그래픽 | Three.js 기반 3D 로우폴리(로블록스풍 블록 스타일), 1인칭 시점 |
| 무기 규모 | 슬롯당 2~3종, 총 8~10종 (주무기/보조무기/근접/유틸리티 4슬롯) |

## 기술 스택

- **클라이언트**: TypeScript + Vite + Three.js
  - 물리/충돌: 자체 캡슐-AABB 충돌 (FPS에 필요한 만큼만, 물리엔진 미사용)
  - 상태 관리: 게임 루프 중심의 단순 구조 (프레임워크 없음)
- **서버 (Phase 5)**: Node.js + TypeScript + WebSocket (Colyseus 또는 ws 직접 사용)
  - 서버 권위(server-authoritative) 위치 검증 + 클라이언트 예측/보간
- **배포**:
  - 클라이언트: GitHub Pages (봇 모드는 서버 없이 동작)
  - 게임 서버: Render / Fly.io 무료 티어 (멀티 단계에서 결정)

## 개발 단계

### Phase 0 — 프로젝트 셋업
- Vite + TypeScript + Three.js 초기화, 기본 씬(맵 바닥, 조명, 스카이박스)
- Pointer Lock(마우스 잠금) 및 기본 렌더 루프

### Phase 1 — FPS 코어 (이동)
- 1인칭 컨트롤러: WASD 이동, 점프, 달리기
- RIVALS식 무브먼트: **대시**(쿨다운), **슬라이드**, 공중 가속
- 로우폴리 1v1 맵 1개 (대칭 구조, 엄폐물, 점프 스팟)
- 캡슐 vs 맵 지오메트리 충돌 처리

### Phase 2 — 전투 시스템
- 무기 프레임워크: 히트스캔(레이캐스트) + 투사체(수류탄 등) 지원
- 초기 무기 8~10종:
  - 주무기: 돌격소총, 샷건, 저격총
  - 보조무기: 권총, 우지
  - 근접: 칼 (우클릭 강공격)
  - 유틸리티: 수류탄, 섬광탄 또는 임팩트 그레네이드
- 데미지/HP(헤드샷 배율), 히트마커, 킬 피드
- HUD: 크로스헤어, 탄약, 체력, 라운드 스코어
- 뷰모델(1인칭 무기 모델) + 반동/스프레이 패턴

### Phase 3 — 1v1 듀얼 게임 루프 (봇 대전)
- 라운드 구조: 스폰 → 전투 → 킬 시 라운드 종료 → 5라운드 선취 승리
- 로드아웃 선택 화면 (라운드 시작 전 무기 선택)
- 봇 AI: 맵 이동(웨이포인트), 플레이어 추적, 조준 오차 기반 난이도(쉬움/보통/어려움)
- 메인 메뉴 → 게임 → 결과 화면 흐름

### Phase 4 — 폴리시
- 사운드(발사, 피격, 발소리, UI), 총구 화염/탄흔/트레이서 이펙트
- 사망 연출, 설정(마우스 감도, 볼륨, FOV)
- 모바일 미지원 안내 또는 최소한의 터치 대응 검토

### Phase 5 — 온라인 멀티플레이 (1v1)
- Node.js WebSocket 게임 서버: 방 생성/코드 입장, 1v1 매치
- 상태 동기화: 클라이언트 예측 + 서버 보간, 틱레이트 20~30Hz
- 서버 측 히트 판정 검증 (기초 수준의 치팅 방지)
- 서버 배포 (Render/Fly.io — 이 단계에서 호스팅 계정 필요)

### Phase 6 — 확장 (선택)
- 추가 모드: FFA, 건 게임
- 추가 무기/맵, 무기 해금·상점 시스템, 킬 이펙트 등 코스메틱

## 리스크 및 참고

- **저작권**: 로블록스/RIVALS의 에셋·이름을 직접 사용하지 않고, "영감을 받은" 오리지널 로우폴리 에셋과 별도 게임명 사용 권장
- **멀티플레이 서버 비용**: 무료 티어는 슬립/트래픽 제한이 있어 데모 수준에 적합. 본격 운영 시 유료 전환 필요
- **성능**: 로우폴리 + 라이트맵 없는 단순 조명으로 저사양 노트북에서도 60fps 목표
